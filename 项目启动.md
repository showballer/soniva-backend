# Soniva Backend — 项目启动说明

本文档说明如何创建 Python 虚拟环境、安装依赖并启动服务。建议 Python **3.10**（与仓库 `Dockerfile` 一致）。

---

## 1. 系统依赖（可选但推荐）

音频相关依赖（`librosa` / `soundfile`）在部分环境下需要系统库。

**macOS（Homebrew）**

```bash
brew install libsndfile ffmpeg
```

**Debian / Ubuntu**

```bash
sudo apt-get update && sudo apt-get install -y libsndfile1 ffmpeg
```

---

## 2. Python 虚拟环境

在项目根目录 `soniva-backend/` 下执行：

```bash
cd /path/to/soniva-backend

# 创建虚拟环境（目录名 venv 可与团队约定）
python3 -m venv venv
```

**激活虚拟环境**

- **macOS / Linux**

  ```bash
  source venv/bin/activate
  ```

- **Windows（cmd）**

  ```cmd
  venv\Scripts\activate.bat
  ```

- **Windows（PowerShell）**

  ```powershell
  venv\Scripts\Activate.ps1
  ```

激活成功后，终端提示符前一般会显示 `(venv)`。

**退出虚拟环境**

```bash
deactivate
```

---

## 3. 安装 Python 依赖

确保已激活虚拟环境，然后在项目根目录执行：

```bash
python -m pip install --upgrade pip
pip install -r requirements.txt
```

---

## 4. 环境变量

在启动前配置 `.env`（数据库、JWT、存储路径等）。若仓库中有 `.env.example`，可复制后修改：

```bash
cp .env.example .env
```

无示例文件时，直接在项目根目录维护 `.env` 即可。主要项见仓库 `README.md`（如 `DATABASE_URL`、`SECRET_KEY`、`LOCAL_STORAGE_PATH`）。

---

## 5. 启动服务

**方式一：Uvicorn（开发常用，支持热重载）**

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

**方式二：以模块方式运行（与 `app/main.py` 中 `__main__` 一致）**

```bash
python -m app.main
```

**方式三：使用项目脚本（需先 `chmod +x start.sh`）**

```bash
./start.sh
```

脚本会 `source venv/bin/activate` 并启动带 `--reload` 的 Uvicorn。

---

## 6. 验证

- API 文档（Swagger）：<http://localhost:8000/docs>
- ReDoc：<http://localhost:8000/redoc>
- 健康检查：<http://localhost:8000/health>

---

## 7. Docker 启动（可选）

若使用容器构建与运行，可参考仓库根目录 `Dockerfile`。镜像内默认命令为：

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 2
```

构建与运行示例（按需挂载 `.env`、数据卷等）：

```bash
docker build -t soniva-backend .
docker run -p 8000:8000 --env-file .env soniva-backend
```
